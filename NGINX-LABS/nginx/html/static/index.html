Nginx Docker Lab

Observe how traffic actually moves.

This stack gives you a local edge proxy, two app upstreams, one API backend, optional TLS, optional WebSockets, and an observability overlay. Use it to inspect routing, forwarding headers, load balancing, rate limits, cache behavior, and HTTP to HTTPS redirects.

Start with /app/, /api/headers, /cache/demo, and /limited/delay/1. Then switch to host-based routing with a Host header such as app.lab.local or api.lab.local.

Path routes

  • /app/ load balances between app-a and app-b.
  • /api/ proxies directly to the API backend.
  • /secure/ adds basic auth. Use student and labpass.
  • /cache/demo shows X-Cache-Status on repeated requests.
  • /ws/ becomes active when the ws profile is running.

Host routes

  • Send Host: app.lab.local to reach the app pool at the root path.
  • Send Host: api.lab.local to reach the API backend at the root path.
  • Use this to compare virtual host routing with path-based routing.

Control loop

  • Test config with docker compose exec nginx nginx -t.
  • Reload with docker compose exec nginx nginx -s reload.
  • Watch logs with docker compose logs -f nginx.
  • Inspect status at /nginx_status from the observability overlay.
